计算机组成原理 - 简答题精解

第一章：计算机系统概论

2、计算机硬件有哪些部件，各部件的作用是什么？

答：计算机的硬件系统由有形的电子器件等构成的，它包括运算器、存储器、控制器、输入输出设备及总线系统组成。

  • 存储器（Memory）：是用来存放数据和程序的部件；
  • 运算器：是对信息进行运算处理的部件，核心是ALU；
  • 控制器：是整个计算机的控制核心。它的主要功能是读取指令、翻译指令代码、并向计算机各部分发出控制信号，以便执行指令；
  • 输入设备：能将数据和程序变换成计算机内部所能识别和接受的信息方式，并顺序地把它们送入存储器中；
  • 输出设备：将计算机处理的结果以人们能接受的或其它机器能接受的形式送出。
4、什么是硬件、软件和固件？什么是软件和硬件的逻辑等价？在什么意义上软件和硬件是不等价的？

答：

  • 计算机硬件（Hardware）：是指构成计算机的所有实体部件的集合，通常这些部件由电路（电子元件）、机械等物理部件组成。
  • 计算机软件（Software）：是指能使计算机工作的程序和程序运行时所需要的数据，以及与这些程序和数据有关的文字说明和图表资料。
  • 固件（Firmware）：是一种介于传统的软件和硬件之间的实体，功能上类似软件，但形态上又是硬件（通常存储在ROM中）。

软件和硬件的逻辑等价含义：

  1. 任何一个由软件所完成的操作也可以直接由硬件来实现（如硬件乘法器）；
  2. 任何一条由硬件所执行的指令也能用软件来完成（如软件模拟浮点运算）。

不等价性：在物理实现、执行速度、成本和灵活性上，软件和硬件是不等价的。硬件通常速度快但成本高、不易修改；软件灵活、成本低但速度较慢。

5、计算机系统按程序设计语言划分为哪几个层次？

答：计算机系统通常划分为以下几个层次：

  1. 微程序级（微指令）
  2. 一般机器级（机器语言）
  3. 操作系统级（系统调用）
  4. 汇编语言级（汇编语言）
  5. 高级语言级（C/C++, Java等）

每一级上都能进行程序设计，且得到下级的支持。

7、常用的计算机性能指标有哪些？

答：主要性能指标包括：

  • 主频 (Clock Rate)：决定了计算机的运行速度，单位MHz/GHz。
  • 字长 (Word Size)：决定了计算机的运算精度、指令字长度、存储单元长度，常见为32位/64位。
  • 运算速度 (MIPS/MFLOPS)：每秒执行的百万条指令数或浮点操作数。
  • 存储容量：内存和外存的容量。
  • 可靠性 (MTBF)：平均无故障时间。
  • 吞吐量 (Throughput)：单位时间内处理的信息量。
  • 响应时间 (Response Time)：从提交请求到获得结果的时间。
8、多媒体的含义是什么？

答：多媒体技术是指能够同时获取、处理、编辑、存储和展示两个以上不同信息类型媒体（如文字、声音、图形、图像、动画、视频）的技术。它强调交互性和集成性。

9、简单描述计算机的层次结构，说明各层次的主要特点。

答：

  • 第0级：硬件逻辑级 - 由门电路、触发器等构成。
  • 第1级：微程序级 - 机器语言指令解释为微指令序列，直接由硬件执行。
  • 第2级：传统机器级 - 程序员可见的机器指令集（ISA）。
  • 第3级：操作系统级 - 提供文件管理、资源调度等服务，扩展了机器功能。
  • 第4级：汇编语言级 - 符号化的机器语言。
  • 第5级：高级语言级 - 面向算法和应用，独立于机器硬件。
10、计算机系统的主要技术指标有哪些？

答：(参见题7，此为类似题)

  • 机器字长：参与运算的数的基本位数。
  • 数据通路宽度：数据总线一次并行传送信息的位数。
  • 主存储器容量：存储空间大小。
  • 运算速度：与主频、CPI（每条指令周期数）等有关。
21、如何区别存储器和寄存器？两者是一回事的说法对吗？

答：不是一回事。

  • 位置：寄存器位于CPU内部；存储器（主存）位于CPU外部。
  • 速度：寄存器速度极快（与CPU同频）；存储器速度较慢。
  • 容量：寄存器数量少，容量极小；存储器容量大。
  • 功能：寄存器用于暂存指令执行过程中的操作数和中间结果；存储器存放完整的程序和数据。
28、指令长度和机器字长有什么关系？半字长指令、单字长指令、双字长指令分别表示什么？

答： 指令长度与机器字长没有固定关系。它可以等于、小于或大于机器字长。

  • 单字长指令：指令长度 = 机器字长。读取一次内存即可获得完整指令。
  • 半字长指令：指令长度 = 1/2 机器字长。读取一次内存可能获得两条指令。
  • 双字长指令：指令长度 = 2倍 机器字长。需要读取两次内存才能获得完整指令。
38、什么叫总线周期、时钟周期、指令周期？它们之间一般有什么关系？

答：

  • 时钟周期 (Clock Cycle)：计算机主频的倒数，最基本的时间单位。
  • 总线周期 (Bus Cycle)：CPU通过总线对存储器或I/O进行一次访问所需的时间。
  • 指令周期 (Instruction Cycle)：取出并执行一条指令所需的全部时间。

关系： 时钟周期 < 总线周期 < 指令周期。
通常一个总线周期包含若干个时钟周期（如4个）。一个指令周期包含若干个机器周期（其中可能包含取指总线周期、执行总线周期等）。

39、说明总线结构对计算机系统性能的影响。

答：

  1. 带宽限制：总线是共享资源，其带宽决定了部件间数据交换的最大速率，可能成为系统瓶颈（冯·诺依曼瓶颈）。
  2. 并发性：单总线结构不支持并发传送；多总线结构（如CPU-Mem总线和I/O总线分离）允许CPU处理和I/O传输并行进行，提高系统效率。
  3. 扩展性：总线标准决定了系统扩展外设的容易程度。
45、什么是中断？中断技术给计算机系统带来了什么作用？

答： 中断是指当计算机执行正常程序时，系统中出现某些异常情况或特殊请求（中断源），CPU 暂停正在执行的程序，保存现场，转去处理所发生的事件（中断服务程序）；处理完毕后，自动返回到原来被中断了的程序继续运行。

作用：

  • 实现主机与外部设备的并行工作（I/O中断）。
  • 处理硬件故障和异常（如掉电、算术溢出）。
  • 实现人机交互。
  • 实现多道程序分时操作（时钟中断）。
46、中断系统为什么要进行中断判优？何时进行中断判优？如何进行判优？

答：

  • 原因：当多个中断源同时提出请求时，CPU一次只能响应一个，必须根据轻重缓急决定响应顺序。
  • 时机：中断判优通常发生在CPU中断查询阶段（指令执行结束前）。
  • 方法：
    • 硬件判优：排队电路（链式查询、中断控制器）。速度快。
    • 软件判优：查询程序按优先级顺序查询。灵活性高但速度慢。

本章核心考点

  • 冯·诺依曼机特点：存储程序、顺序执行、二进制。
  • 计算机系统层次：硬件 -> 微程序 -> 机器语言 -> 操作系统 -> 汇编 -> 高级语言。
  • 性能指标：主频、字长、CPI、MIPS、带宽、响应时间。
  • 硬件五大部件：运算器、控制器、存储器、输入、输出。

第二章：数据的表示和运算

6、解释如下概念：ALU, CPU, 主机和字长。

答：

  • ALU (Arithmetic Logic Unit)：算术逻辑单元，负责执行加减乘除、与或非等运算。
  • CPU (Central Processing Unit)：中央处理器，核心部件，包括ALU、控制器（CU）和寄存器组。
  • 主机：计算机硬件的核心部分，由CPU和主存储器（Memory）组成。
  • 字长：CPU内部数据通路一次能并行处理的二进制位数，标志着计算机的精度和处理能力。
12、海明校验码的编码规则有哪些？

答：海明码（Hamming Code）是一种多重奇偶校验码，具有纠错能力。

  1. 校验位位置：校验位 $P_i$ 位于 $2^{i-1}$ 的位置（即第1, 2, 4, 8...位）。
  2. 位数关系：若有 $k$ 个数据位，$r$ 个校验位，需满足 $2^r \ge k + r + 1$。
  3. 校验分组：第 $i$ 组包含所有二进制位置号中第 $i$ 位为1的位。$P_i$ 负责校验该组的奇偶性。
13、简述 CRC 码的纠错原理。

答：CRC（循环冗余校验）基于模2除法。

  • 发送端：将数据左移，除以生成多项式 $G(x)$，将得到的余数（FCS）拼接到数据后发送。
  • 接收端：将接收到的数据除以同样的 $G(x)$。
    • 若余数为0，说明无错。
    • 若余数不为0，说明有错。不同的余数对应不同的出错位（在单位错假设下），可通过查表或计算确定出错位置并纠正。
14、运算器由哪几部分组成？

答：

  1. ALU：算术逻辑运算单元（核心）。
  2. 通用寄存器组（GPRs）：如AX, BX等，存放操作数和中间结果。
  3. 暂存寄存器：锁存输入数据。
  4. 累加器（ACC）：特殊的寄存器。
  5. 程序状态字寄存器（PSW）：存放运算标志（溢出、进位、零等）。
  6. 内部总线：连接各部件。

本章核心考点

  • 原码/反码/补码/移码定义及转换。
  • 补码运算：将减法变加法，符号位参与运算。
  • 定点数 vs 浮点数：范围与精度的权衡。
  • 校验码：奇偶校验、海明码（纠错）、CRC（检错）。
  • 溢出判断：双符号位法、进位判别法。

第三章：存储器系统

15、主存储器有哪些性能指标？它们的含义是什么？

答：

  1. 存储容量：存储单元数 × 字长。
  2. 存取时间 (Access Time, $T_A$)：启动读/写到操作完成的时间。
  3. 存取周期 (Memory Cycle Time, $T_{MC}$)：连续两次独立访问所需的最小间隔时间。通常 $T_{MC} > T_A$。
  4. 存储带宽 (Bandwidth)：单位时间内存储器存取的信息量（例如 B/s）。
16、主存的基本组成有哪些部分？各部分主要的功能是什么？

答：

  • 存储体：由存储元阵列组成，实际存放二进制信息。
  • MAR (Memory Address Register)：地址寄存器，锁存要访问的单元地址。
  • MDR (Memory Data Register)：数据寄存器，暂存读出或写入的数据。
  • 地址译码器：将MAR中的地址转换为选通信号，选中特定单元。
  • 读写控制电路：接收读/写命令，控制时序。
17、静态 MOS 存储元、动态 MOS 存储元各有什么特点？

答：

  • SRAM (静态)：利用双稳态触发器存储信息。速度快，不需要刷新，但集成度低，功耗大，成本高。常用作Cache。
  • DRAM (动态)：利用电容电荷存储信息。集成度高，成本低，功耗小，但速度较慢，且需要定时刷新（因为电容漏电）。常用作主存。
18、什么是刷新？为什么要刷新？有哪几种常用的刷新方式？

答：

  • 定义：对DRAM的存储单元进行读出并恢复电荷的操作。
  • 原因：DRAM依靠电容存储电荷，电荷会随时间泄漏，若不及时补充（刷新），信息将丢失。
  • 方式：
    1. 集中刷新：在规定的刷新周期内，安排一段时间集中刷新所有行，此时停止读写（死区）。
    2. 分散刷新：将存取周期延长，前一半读写，后一半刷新。无死区，但存取周期变长。
    3. 异步刷新：将刷新操作分散在整个刷新周期内进行。效率高，死区短。
19、简要说明提高存储器速度有哪些措施？

答：

  1. 采用更高速的器件：如DDR SDRAM。
  2. Cache技术：利用SRAM作为高速缓冲。
  3. 多体交叉存储 (Interleaved Memory)：并行访问多个存储模块，提高带宽。
  4. 双端口存储器：允许两路同时访问。
20、Cache 有哪些特点？

答：

  • 速度：比主存快5-10倍，接近CPU速度。
  • 容量：比主存小得多。
  • 内容：存放主存中最活跃的数据副本（局部性原理）。
  • 透明性：对程序员透明，由硬件自动管理。
22、存储器的主要功能是什么？为什么要把存储系统分成若干个不同层次？主要有哪些层次？

答：

  • 功能：存储程序和数据。
  • 分层原因：为了解决容量、速度和价格之间的矛盾（没有一种存储器能同时满足容量大、速度快、价格低）。
  • 层次：
    • Cache-主存层：解决速度匹配问题（对程序员透明）。
    • 主存-辅存层（虚拟存储器）：解决容量问题。
23、说明存储周期和存取时间的区别。

答：

  • 存取时间 ($T_A$)：从发出读命令到数据稳定出现在输出端的时间。
  • 存取周期 ($T_{MC}$)：存储器进行一次完整的读写操作后，内部电路恢复原状并准备好下一次请求所需的总时间。
  • 关系：$T_{MC} > T_A$。
34、指令和数据都存放在主存，如何识别从主存储器中取出的是指令还是数据？

答：

  1. 时间维度：在取指周期（Fetch Cycle）取出的是指令；在执行周期（Execution Cycle）取出的是数据。
  2. 来源维度：指令地址来源于PC（程序计数器）；数据地址来源于指令中的地址码或运算结果。
48、磁表面存储器的特点有哪些？

答：

  • 优点：存储密度高、容量大、单位成本低、非易失性（断电不丢失）、可脱机保存。
  • 缺点：存取速度慢（毫秒级）、结构复杂（机械）、对环境要求高（防尘防震）。

本章核心考点

  • 存储层次：Cache - 主存 - 辅存。
  • SRAM vs DRAM：速度、刷新、集成度。
  • 存储扩展：位扩展（数据线）、字扩展（地址线）。
  • Cache原理：局部性原理，映射方式（直接、全相联、组相联）。
  • 存储器性能：存取时间 vs 存取周期。

第四章：指令系统

1、冯·诺依曼型计算机的基本特点是什么？

答：

  1. 存储程序：程序和数据以二进制形式同等地位存放在存储器中。
  2. 顺序执行：CPU自动从内存取出指令并执行，默认按顺序递增PC。
  3. 五大部件：运算器、控制器、存储器、输入设备、输出设备。
  4. 以运算器为中心（现代通常以存储器为中心）。
24、指令格式设计的准则有哪些？

答：

  • 短：尽量缩短指令长度，节省存储空间和取指时间。
  • 多：要有足够的指令类型和操作码位数。
  • 整：指令长度应是机器字长的整数倍，便于读取。
  • 活：寻址方式丰富灵活。
25、指令是灵活多变的，体现在哪些方面？

答：

  • 操作码可变：如扩展操作码技术。
  • 指令长度可变：单字长、双字长等。
  • 寻址方式多样：立即、直接、间接、变址、相对等。
  • 地址码个数可变：零地址、一地址、二地址等。
26、试比较基址寻址和变址寻址的异同。

答：

  • 相似点：形式上都是 $EA = (R) + A$。
  • 不同点：
    • 基址寻址：基址寄存器内容由OS管理（固定），形式地址A可变。用于程序重定位。
    • 变址寻址：形式地址A固定（基准），变址寄存器内容由用户改变。用于数组处理、循环操作。
27、堆栈是什么？它有什么特点？功能有哪些？

答：

  • 定义：一组顺序的存储单元，按“先进后出”（LIFO）原则存取。
  • 特点：只能在栈顶（Top）进行插入（Push）和删除（Pop）。栈顶由栈指针（SP）指示。
  • 功能：
    • 支持零地址指令。
    • 保存子程序调用/中断的返回地址和现场。
    • 参数传递。
29、计算机进行程序控制工作的基本原理是怎样的？

答：即“取指-执行”循环。

  1. 取指令：PC -> MAR -> Mem -> MDR -> IR。
  2. 分析指令：指令译码，形成有效地址。
  3. 执行指令：发出控制信号，完成操作。
  4. 下条指令：PC + 1，重复上述步骤。
31、微程序控制的基本思想是什么？

答：仿照程序设计的思想。将机器指令分解为微操作序列，每个微操作由微指令控制。一组微指令构成微程序。将微程序存放在控制存储器（CM）中。执行机器指令时，就是执行对应的微程序。

32、说明微指令和机器指令的区别？

答：

  • 机器指令：硬件和软件的接口，由操作码和地址码组成，功能较复杂。
  • 微指令：硬件层面的控制命令，由微命令和下地址组成，功能简单（如寄存器传送）。
  • 关系：一条机器指令对应一段微程序（多条微指令）。

本章核心考点

  • 指令格式：操作码 + 地址码。
  • 寻址方式：立即、寄存器、直接、间接、基址（OS用）、变址（循环用）、相对（跳转用）。
  • CISC vs RISC：复杂 vs 精简。
  • 堆栈操作：先进后出，支持子程序嵌套。

第五章：中央处理器

30、控制器的基本功能是什么？基本组成部件包括哪些？

答：

  • 功能：取指令、分析指令、执行指令、控制程序流向、处理异常。
  • 组成：
    • PC：程序计数器。
    • IR：指令寄存器。
    • ID：指令译码器。
    • CU：控制单元（时序发生器、微操作信号发生器）。
33、控制器有哪些几种控制方式？各自有什么特点？

答：

  1. 同步控制：统一时钟，步调一致。电路简单，但速度受限于最慢的操作。
  2. 异步控制：应答方式（握手）。无统一时钟，速度快，适应性强，但电路复杂。
  3. 联合控制：大部分同步，小部分异步（如存储器读写）。
35、什么是微指令和微操作？微程序和机器指令有何关系？微程序和程序之间有何关系？

答：(参见题31/32)

  • 微操作：计算机中最基本的原子操作（如 A->B）。
  • 微指令：控制一组微操作的命令字。
  • 微程序 vs 程序：
    • 微程序位于控制存储器（固件），解释机器指令，用户不可见。
    • 程序位于主存（软件），由机器指令组成，用户编写。
36、比较水平微指令和垂直微指令的优缺点。

答：

  • 水平型：字长长，并行能力强，执行速度快，编写困难（由于太底层）。
  • 垂直型：字长短，类似机器指令，编写容易，但并行能力差，速度慢。

本章核心考点

  • CPU功能：指令控制、操作控制、时间控制、数据加工。
  • 控制方式：同步、异步。
  • 微程序控制：微指令解释机器指令，存储在CM中。
  • 流水线技术：提高吞吐率，需解决相关性（数据、结构、控制冲突）。

第六章：总线系统

3、什么是总线？以总线组成计算机有哪些组成结构？

答：

  • 定义：连接多个部件的信息传输线，是共享的公共通道。
  • 结构：
    • 单总线：CPU、内存、I/O挂在同一条总线上。
    • 双总线：CPU-内存总线 + I/O总线。
    • 多总线：增加DMA总线、PCI总线等。

本章核心考点

  • 总线分类：片内、系统、I/O。
  • 总线仲裁：集中式（链式、计数器、独立请求） vs 分布式。
  • 总线带宽计算：频率 × 宽度 / 8。
  • 总线操作阶段：申请、寻址、传输、结束。

第七章：输入输出系统

37、比较单总线、双总线和多总线结构的性能特点。

答：

  • 单总线：结构简单，成本低，但带宽争用严重，不支持并行。
  • 双总线：分离了存储器流量和I/O流量，提高了效率。
  • 多总线：针对不同速度设备使用不同总线（如高速PCIe，低速USB），效率最高，结构最复杂。
40、接口电路在系统结构中的作用是什么？

答：接口（Interface）是主机和外设之间的桥梁。

  • 作用：
    • 数据缓冲（解决速度差异）。
    • 信号转换（数字/模拟，电平转换）。
    • 地址译码（设备选择）。
    • 状态/控制信息传递。
41、接口电路应具备哪些基本功能？

答：(同上) 选址、缓冲、控制、状态反馈、数据转换。

42、外部设备在系统中如何编址，如何与主机连接？

答：

  • 编址方式：
    1. 统一编址（存储器映射）：I/O端口占用内存地址空间。指令统一。
    2. 独立编址（I/O映射）：I/O端口有独立地址空间。需专用指令（IN/OUT）。
  • 连接方式：通过接口电路连接到总线。
43、什么是 I/O 组织方式？有哪些几种 I/O 组织方式？各自的特点是什么？

答：

  1. 程序查询：CPU忙等查询状态，效率低，硬件简单。
  2. 程序中断：CPU可与外设并行，适用于中低速设备。
  3. DMA (直接存储器访问)：硬件控制内存和外设直接交换，CPU仅在开始和结束参与。适合高速块传输。
  4. 通道 (Channel)：专用I/O处理器，执行通道程序。
44、查询方式和中断方式的主要异同点是什么？

答：

  • 相同：都需要CPU执行指令来传送数据（中断方式下是服务程序）。
  • 不同：
    • 查询：CPU被动等待，串行工作。
    • 中断：CPU主动，并行工作，效率高。
47、外部设备有哪些主要功能？可以分为哪些大类？各中有哪些典型设备？

答：

  • 人机交互：键盘、鼠标、显示器。
  • 存储：磁盘、光盘。
  • 机机通信：网卡、调制解调器。

本章核心考点

  • I/O控制方式：程序查询（串行）、中断（并行）、DMA（高速块传输）、通道。
  • 中断过程：请求 -> 响应（中断隐指令）-> 服务 -> 返回。
  • DMA特点：窃取总线周期，CPU仅干预开始结束。
  • 接口功能：缓冲、转换、控制、状态。

其他 / 综合

11、试计算采用 32×32 点阵字形的一个汉字字形占多少字节？存储 6763 个 16×16 点阵以及 24×24 点阵字形的汉字库各需要多少存储容量？

答：

  • 32x32点阵：(32*32)/8 = 1024/8 = 128 字节。
  • 16x16汉字库：6763 * (16*16/8) = 6763 * 32 = 216,416 字节。
  • 24x24汉字库：6763 * (24*24/8) = 6763 * 72 = 486,936 字节。
